・CANoe本体　：Vector CANoe VN1630

■Ph6リプログ CANoeシナリオ使用方法について
・APL起動要求を契機にCANoeシナリオを起動する方法
　以下の手順でCANoeシナリオを実行します。
　1．「https://1st.aubswdev.com/svn/dev06/rprg-cp/trunk/ESC0154_20rprg_A0_1(U2A8)/technology/design/07_SIT/WP020_021_ソフトウェア統合テスト仕様・結果/_environment/common/Phase6CANoe」
　　 を任意の場所(ただしフルパスで128文字以内)にチェックアウトします。
　2. 「SBL.csv」を編集し、書換え対象領域毎の消去ルーチンに関する各種情報を記述します。
　3. 「Nonce.csv」を編集し、Nonce値を記述します。
　4. 「applicationSoftwareFingerprint.csv」を編集し、書き換え対象領域毎のFingerprint値を記述します。
　5. 「APL.csv」を編集し、書換え対象領域毎の書き換えファイルに関する各種情報を記述します。
　6． CANoeを起動します。
　7． CANoeのメニューバーの「ファイル　→　コンフィグレーションを読み込む」を順にクリックし、
　　  開いたウィンドウに本書と同階層にある「Reprog.cfg」を選択して開くをクリックします。
　8． CANoeの開始ボタンをクリックします。
　9． キーボードの「a」キーを押下します。
 10． CANoeがSID 0x10(ProgrammingSession)のポジティブレスポンスを受信すると、
　　　自動でCANoeシナリオに沿ったデータ送受信が行われます。

■外部設定情報について
以下の情報はあらかじめ外部設定情報ファイル「Reprog_Ext.ini」に設定しておくことで、
CANoeシナリオ起動時に読み出して使用します。

・通信設定用の診断記述ファイル名(64バイト)
・DLL用の診断記述ファイル名(64バイト)
・Nonce値を記述したファイル名(128バイトまで)
　⇒ここで指定したファイルの中にはNonce値を1個記載可能
・applicationSoftwareFingerprintを記述したファイル名(128バイトまで)
  ⇒ここで指定したファイルの中にはFingerprint値を最大10個記載可能
・CANoeシーケンス停止の契機となるSID(指定したSIDの送信直前から、それ以降の処理を停止)
・CANoeシーケンス停止の契機となるSIDのSub-function
・CANoeシーケンス停止の契機となるSIDのタイミング情報( 0x0=SBL / 0x1=APL)
・SID 0x10(extendedDiagnosticSession)送信時のSPRMIB設定( 0x0=SPRMIB無し / 0x1=SPRMIB有り )
・SID 0x22の実行制御を設定( 0x0=スキップ / 0x1=実行 )
・SID 0x3Eのアドレス種別を設定( 0x0=物理アドレス / 0x01=機能アドレス )
・SID 0x3E送信時のSPRMIB設定( 0x0=SPRMIB無し / 0x1=SPRMIB有り )
・SID 0x27のSeed値からKey値を生成する際のツール認証キー
・SID 0x31(check)のチェック対象を設定( 0x0=チェックしない / 0x1=チェックする )
  ⇒ここで0x1( チェックする )を指定した checkTypeIdentifier(signature / CMAC / CRC )数分、SID0x31(check)を送信する。
・リセット時のSIDの指定( 0x0=SID 0x10[DiagnosticSessionControl] / 0x1=SID 0x11[ECUReset])
・消去書込みルーチン情報ファイルのファイル名(128バイトまで)
  ⇒ここで指定したファイルの中には以下の情報全てを1セットとし、10セットまで記載可能。
　　消去書込みルーチンの仮想アドレスの「サーバの書換え対象領域番号」(1バイトで表すことのできる範囲まで)
　　消去書込みルーチンの仮想アドレスの「物理アドレス」(4バイトで表すことのできる範囲まで)
　　消去書込みルーチンの転送するデータのサイズ(4バイトで表すことのできる範囲まで)
　　消去書込みルーチンの非圧縮/圧縮の指定(0x0=非圧縮 / 0x1=圧縮 / 0x2=差分)
　　消去書込みルーチンの平文/暗号文の指定(0x0=平文   / 0x1=暗号文)
    消去書込みルーチンのsignatureを記述したファイル名(128バイトまで)
　　消去書込みルーチンのCMACを記述したファイル名(128バイトまで)
　　消去書込みルーチンのCRCを記述したファイル名(128バイトまで)
　　消去書込みルーチンの処理順番(0x0～0xf　，省略した場合は上から順に0x0からの連番が適用される)
・書換えファイル情報ファイルのファイル名(128バイトまで)
　⇒ここで指定したファイルの中には以下の情報全てを1セットとし、10セットまで記載可能。
　　書換えファイルの仮想アドレスの「サーバの書換え対象領域番号」(1バイトで表すことのできる範囲まで)
　　書換えファイルの仮想アドレスの「物理アドレス」(4バイトで表すことのできる範囲まで)
　　書換えファイルの転送するデータのサイズ(4バイトで表すことのできる範囲まで)
　　書換えファイルの非圧縮/圧縮の指定(0x0=非圧縮 / 0x1=圧縮 / 0x2=差分)
　　書換えファイルの平文/暗号文の指定(0x0=平文   / 0x1=暗号文)
    書換えファイルのsignatureを記述したファイル名(128バイトまで)
　　書換えファイルのCMACを記述したファイル名(128バイトまで)
　　書換えファイルのCRCを記述したファイル名(128バイトまで)
　　書換えファイルの処理順番(0x0～0xf　，省略した場合は上から順に0x0からの連番が適用される)
